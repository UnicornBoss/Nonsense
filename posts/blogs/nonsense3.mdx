# Keynote

1. React's APIs are rooted primarily in <span className="react">design</span> principles, not programming.
2. When <span className="react">designers</span> and <span className="react">programmers</span> speak different languages, the user experience gets lost in translation.
3. When you use React, it feels like you're directly authoring the user experience.
4. Truly great <span className="react">developer experience</span> does not sacrifice <span className="react">user experience</span> -- it enables it.
- North star, to make it easier to build great user experience.

## Suspense

Read <span className="react">data</span> over the network as easily as <span className="react">props</span> or <span className="react">state</span>.

阅读网络上的数据，就像是props或state一样。

```jsx
function Header() {
  const { data, isLoading } = useData(...);

  if (isLoading) {
    return /* ??? */;
  }

  return (...);
}
```
经典的React模式如上，获取数据和加载状态是放在一个位置的，但React团队认为这应该是两件事，对于一个组件来说，他只需要处理他所需的数据就好，而不需要管理加载状态（当然也是从设计师的角度）。

```jsx
<Suspense fallback={<Spinner />}>
  <Header />
  <Suspense fallback={<ListPlaceholder />}>
    <List pageId={pageId} />
  </Suspense>
</Suspense>
```
使用Suspense的React模式如上，

不需要对数据获取部分的代码做任何改动，只需要关注在哪里需要添加或删除Suspense组件

<span className="react">Client</span>: Date fetching via compatible libraries, like Relay, SWR, React Graph.

<span className="react">Server</span>: SSR support with streaming HTML, <span className="react">Suspense</span>

## React 18

### Concurrent rendering

A <span className="react">behind-the-scenes</span> capability in React that powers features like Suspense.
一种React的幕后功能，比如像Suspense功能。

You focus on <span className="react">what</span> you want your app to feel like, and React takes care of <span className="react">how</span> to deliver that experience.
你聚焦于你想要的应用程序体验，React会负责如何提供这种体验。

There is no concurrent <span className="react-red">mode</span> --
there are only concurrent <span className="react">features</span>.
换句话说，以后不会有并发模式，只会有并发功能。

主要讲了升级React 18 会带来什么。
